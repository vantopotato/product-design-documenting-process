## 🎯 为什么需要这个 Skill？ (解决的痛点)

在与 Claude Code 协作进行复杂的业务设计或代码重构时，开发者通常会面临两个致命问题：**“需求幻觉”**与**“大面积返工”**。本框架通过强制性的三阶段流水线，针对性地解决了这些痛点。

### 1. 强制反刍：击碎 AI 幻觉
*   **痛点**：用户输入需求后，AI 往往会立刻开始“狂奔”，生成大量看起来正确但实际上理解偏离的逻辑方案（幻觉）。
*   **本 Skill 的解法**：通过 `Pre-flight Hook` 和 `第一阶段：逻辑对齐`，强制 AI 必须先“消化”并复述用户需求。在交互过程中，这迫使开发者与 AI 共同完成对需求合理性的**反刍**。
*   **效果**：在方案写出第一行代码前，偏差就已经被纠正。

### 2. 步骤拆分：消除灾难性返工
*   **痛点**：一次性交给 AI 一个大任务，产出结果后如果发现核心逻辑错了，整个上下文和代码都需要大范围重写（返工）。
*   **本 Skill 的解法**：将任务硬性拆分为 **“对齐 -> 细化 -> 交付”** 三个阶段。每一阶段都有 `Transition Hook`（转换确认钩子）。
*   **效果**：将返工的概率从“任务末尾”提前到了“任务早期”，将大面积的代码重构转化为细微的参数调整。

## 💎 框架价值

- **从“黑盒对话”变为“白盒协作”**：你清楚地知道 AI 现在的思考进度。
- **让 AI 拥有“慢思考”能力**：借鉴了类似 OpenAI o1 的推理思路，但通过工作流在本地实现。
- **自适应业务场景**：只需通过 `quick-setup.sh` 配置，即可让这套严谨的思考模型适配你的具体业务场景。


## 🛠️ 快速上手

### 方式 A：自动化配置 (推荐)
1. 克隆本仓库：`git clone <repository-url>`
2. 运行脚本：`bash quick-setup.sh`
3. 按照提示输入您的业务背景。
4. 脚本会为您生成定制化的 `CLAUDE.md`。

### 方式 B：手动快速集成
将本仓库中的 `CLAUDE_TEMPLATE.md` 内容复制并粘贴到你项目根目录的 `CLAUDE.md` 文件中，通过搜索 `[占位符]` 手动编辑。


## ⚙️ 核心机制：工作流钩子 (Hooks)

本框架最大的特色是引入了类似 Webhook 的执行逻辑：

- **Pre-flight Hook**：在 AI 开始工作前，强制检查输入质量。
- **Alignment Hook**：在阶段转换时，强制 AI 停下来与你同步共识。
- **Safety Hook**：当涉及敏感操作时，自动触发风险评估逻辑。
